Ferramenta de proteção à criança

O que é o child-protection?

Child protection é uma biblioteca javaScript que auxilia desenvolvedores a previnir abuso de crianças que estão em contato com seus produtos. A ferramenta dispõe de diverssos metodos que podem ser utilizados na construção das aplicações. Aferramenta oferece uma camada de proteção com cripitografia de ponta a ponta.

Instalação

Requerimentos:

  • Familiaridade com linha de comando
  • Ter instalado o Node.js versão 16.0 ou superior

Certifique-se de ter uma versão atualizada do Node.js instalada e execute o seguinte comando na linha de comando

                        
                            npm install child-protection

Este comando irá instalar o pacote child-protection, a ferramenta oficial de andaimes da biblioteca child-protection. Você receberá prompts para vários recursos opcionais, como TypeScript e suporte a testes

uso

A estratégia de autenticação é construída da seguinte forma:

                        
                            import 'child-protection' from 'child-protection';

                            const protection = new child-protection(options, verify);
                        
                    
                        
                            var JwtStrategy = require('passport-jwt').Strategy,
                            ExtractJwt = require('passport-jwt').ExtractJwt;
                            var opts = {}
                            opts.jwtFromRequest = ExtractJwt.fromAuthHeaderAsBearerToken();
                            opts.secretOrKey = 'secret';
                            opts.issuer = 'accounts.examplesoft.com';
                            opts.audience = 'yoursite.net';
                            passport.use(new JwtStrategy(opts, function(jwt_payload, done) {
                                User.findOne({id: jwt_payload.sub}, function(err, user) {
                                    if (err) {
                                        return done(err, false);
                                    }
                                    if (user) {
                                        return done(null, user);
                                    } else {
                                        return done(null, false);
                                        // or you could create a new account
                                    }
                                });
                            }));

Para combater efetivamente, você deve combinar validação de entrada com higienização de saída. Usar um ou outro não é suficiente; você deve aplicar ambos! Além disso, validações simples como comprimento de corda não são tão eficazes

A higienização da saída (também conhecida como Filtragem de Ouput) é o responsável.secure-filters

Para santizar adequadamente a saída, você precisa ser sensível ao contexto em que os dados estão sendo produzidos. Por exemplo, se você quiser colocar o texto em um documento HTML, você deve html-escape do texto.

Mas e os contextos CSS ou JavaScript? Você não pode usar o filtro html-escape; um método diferente de fuga é necessário. Se o filtro não corresponder ao contexto, é possível que os navegadores interpretem mal o resultado, o que pode levar a ataques XSS!

Funções

  • html(valor) - Higieniza contextos HTML usando codificação de entidades.
  • js(valor) - Higieniza os contextos de sequência javaScript usando codificação de barras invertidas.
  • jsObj(valor) - Higieniza os literais JavaScript (números, strings, booleanos, arrays e objetos) para inclusão em um contexto de script HTML.
  • jsAttr(valor) - Higieniza os contextos de string JavaScript em um atributo HTML usando uma combinação de codificação de entidade e backslash.
  • uri(valor) - Higieniza os contextos uri usando codificação percentual.
  • css(valor) - Higieniza os contextos CSS usando codificação de barras invertidas.
  • style(valor) - Higieniza contextos CSS em um atributo estilo HTML

Extensão de proteção para navegadores

O que é a extensão child protection?

O nosso plugin licenciado e oficializado utiliza tecnologias para o monitoramento de palavras chaves de estimulo abusivo sexual. Esses pontos chaves são ciptografados, gerando um relatorio automatico mandado diretamente para o seu email, pai e/ou mãe responsável. Nada geral, seja avulso ou cotidiano das conversas paralelas das crianças é monitorado por nós ou pelo algoritmo do puglin. De forma puramente analitica de vocabulario e margens duvidosas, junto com o nosso banco de dados, o puglin faz uma varredura desse determinado momento e direciona para o email cadastrado, sinalizando um possível abuso que seu filho ou filha pode estar sofrendo, sem o mesmo sequer saber. Nosso puglin tambem utiliza a tecnologia de geometria computacional e detecção de pixels para reconhecer arquivos que contenham nudez. As fotos são catalogadas, já os vídeos, têm capturas tiradas com base nos mesmos parâmetros, de forma a identificar seu conteúdo de forma mais fácil, gerados até mesmos por anúncios repentinos em diversos sites.
